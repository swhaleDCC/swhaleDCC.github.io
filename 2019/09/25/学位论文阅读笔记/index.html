Android恶意代码检测技术的研究与实现

肖强-北京邮电大学-2017
[链接](https://kns.cnki.net/KCMS/detail/detail.aspx?dbcode=CMFD&dbname=CMFD201801&filename=1017290472.nh&uid=WEEvREcwSlJHSldRa1FhdXNXaEhobmcrajlaWWVITzdybTVNaGNOMm9tcz0=$9A4hF_YAuvQ5obgVAqNKPCYcEjKensW4IQMovwHtwkF4VYPoHbKxJw!!&v=MDA0MjIyNkdiR3hIdFhMclpFYlBJUjhlWDFMdXhZUzdEaDFUM3FUcldNMUZyQ1VSTE9lWitkc0Z5bmtWNzdMVkY=)

摘要：随着移动互联网的高速发展,搭载Android操作系统的智能手机已经成为人们生活中不可或缺的一部分。由于Android系统的开放性,应用程序开发者可以随意在Android平台上开发和发布应用程序,并且无需进行身份认证,因此Android系统成了恶意程序集中爆发的一个平台。静态检测虽然效率较高,但是难以应对代码加密混淆和多态变形技术。动态检测技术因其通过实际运行程序来获取应用行为特征,可以有效弥补静态检测所带来的局限性。影响动态检测方法检出率的关键技术有两个:一个是应用行为监控技术;另一个是软件行为触发技术;对于应用行为的监控,现有的技术已较为丰富,通过此类技术,可以在多个层次,多个维度对应用行为进行监控,因此行为触发成为了动态分析的重点和难点。本文首先介绍了当前Android恶意代码检测技术的研究现状,分析了当前在动态检测方法上的不足。从如何提高恶意行为触发率角度出发,结合当前Android平台下自动化检测技术,提出了基于双重遍历的行为触发方法。该方法由深度界面遍历以及组件遍历两部分构成。深度界面遍历以基于控件的深度遍历算法,重新构建遍历规则,并且在遍历的过程中会对界面进行分析,对控件进行筛选以此减少重复操作提高遍历的效率。组件遍历用以对应用程序的Service、Receiver进行遍历,以期触发可能隐藏于Service和Receiver中的恶意行为。同时,为实现Android恶意应用的行为监控,对基于Zygote注入的HOOK技术方案进行了研究与实现,通过对应用程序框架层以及核心库中API的劫持,获取应用行为特征。在行为分析阶段,使用机器学习中的分类算法-支持向量机对应用进行检测判别。实验分析结果表明,本文所提出的行为触发方案能够提高程序运行过程的函数覆盖率和恶意行为触发率,进而在一定程度上提高恶意行为检测效果。

静态检测:

  • 优点:效率高
  • 缺点:难以应对代码加密混淆和多态变形技术

动态检测:

  • 通过实际运行程序来获取应用行为特征
  • 影响检出率的关键技术1:应用行为监控技术
  • 影响检出率的关键技术2:软件行为触发技术(重点和难点)

Android软件恶意行为检测技术研究

曲家兴-哈尔滨工程大学-2017
[链接]( https://kns.cnki.net/KCMS/detail/detail.aspx?dbcode=CDFD&dbname=CDFDLAST2018&filename=1018083097.nh&uid=WEEvREcwSlJHSldRa1FhdXNXaEhobmcrajkrUVcxODJ0WDRFYjkxU2NMRT0=$9A4hF_YAuvQ5obgVAqNKPCYcEjKensW4IQMovwHtwkF4VYPoHbKxJw!!&v=MDY3ODFGMjZGck93SGRIRnFKRWJQSVI4ZVgxTHV4WVM3RGgxVDNxVHJXTTFGckNVUkxPZVorZHNGeURuVzd6T1Y= )

摘要：随着移动互联网的普及,搭载Android系统的智能终端因其出众的开放性、丰富的第三方应用软件、友好的操作界面和良好的用户体验等显著优势,给人们的生活带来了诸多便捷,得到了广泛的应用,成为人们生活中不可或缺的重要工具,其在移动端的普及率已达到世界第一。Android系统智能终端上可以安装丰富的应用,如社交、网上银行、电子支付等软件,这些软件上存有大量的用户信息,具有很强的应用价值,一旦泄露将给用户带来难以估量的经济损失。然而,Android系统的开放性也使得针对其开发的恶意软件的数量逐年增加,各种非法恶意软件给用户带来了严重的安全隐患。目前,针对Android平台的恶意软件已经给恶意行为检测技术带来了一定的挑战:一方面,Android系统的开源性、用户权限复杂、应用软件的数字签名无权威第三方认证这些特性,导致软件恶意行为具有复杂性和隐藏性,传统的恶意行为检测技术难以将其识别;另一方面,搭载Android系统的移动终端平台,能源和计算能力受限,传统PC领域的软件恶意行为检测技术和方案难以移植到Android系统上。因此,本文的研究以搭载Android系统的智能终端为对象,提出了若干适用于Android系统的软件恶意行为检测技术和方法,相比较之前的方法,本论文的检测效率取得了进一步提升。概括起来,本文的主要贡献如下:第一,提出了面向行为规则的Android软件恶意行为检测机制。首先,使用基于二元映射编码的软件行为描述语言,将Android软件再映射为按执行时序排列的二元组。然后,采用马尔科夫链构建行为转移概率图。接着,提出一套行为约简规则集,用于从行为转移概率图中提取恶意行为的“指纹”,根据行为指纹判别软件是否具有恶意行为;最后,通过实验验证的方式评价了本方法的有效性和可用性。第二,提出了多策略组合的Android软件恶意行为检测方案。创新地设计了一个四层恶意行为检测架构,包括动静方式相结合的特征提取模块、基于多策略的初级检测算法模块、基于神经网络的多策略权重学习模块和基于投票策略的恶意行为判别模块。首先,采用动静结合的策略从多个维度收集软件的行为特征,并同时将支持向量机、贝叶斯网络、k-近邻等知名机器学习技术作为初级检测机制对其进行检测;然后,在迭代训练阶段通过采用神经网络技术修正各初级检测机制的决策权重,提高系统对恶意行为的检测能力;最后,通过实验验证的方式评价了本方法的有效性。第三,提出了支持任务迁移的轻量级Android软件恶意行为检测方法。首先,在Android手机客户端提出一种电量消耗敏感的恶意软件检测模型,构造程序代码CPU运行周期与电量消耗的映射关系,在程序解释执行时,通过比对预判能耗与程序的“标准”能耗的关系来标记相关进程是否存在“疑似恶意行为”;其次,根据代码的特性及运行时网络状况,采用任务“捆绑”策略将相应的任务在线迁移至云端,该策略在任务迁移时同时考虑了用户的体验及任务迁移所带来的附加能耗;然后,在云端通过采用虚拟机自省技术来跟踪迁移其上代码的“运行轨迹”。最后,通过实验证明,提出的检测架构在保证高检测率的同时,检测的时间和能量开销都大幅降低。总体而言,本文就针对Android系统恶意行为检测技术进行了系统的研究,针对静态恶意行为检测和动态恶意行为检测,创新地设计了一套针对恶意行为转化概率图的检测约简规则集,在降低误报率和漏报率的同时,大幅提升检测效率。此外,结合云计算技术的兴起,提出基于云平台的轻量级Android软件恶意行为检测方法,创新地将恶意行为引起的能耗波动作为检测元素,利用将疑似恶意行为迁移到云端运行与检测的机制,大幅提高Android终端安全性的同时,通过采用捆绑迁移并行执行的策略,兼顾了用户的体验,对未来Android系统软件恶意行为检测的继续开展具有广阔的研究价值和研究空间。

Android恶意代码检测技术研究

吴迪-解放军信息工程大学-2017
[链接]( https://kns.cnki.net/KCMS/detail/detail.aspx?dbcode=CMFD&dbname=CMFD201801&filename=1018702340.nh&v=MjcyMThNMUZyQ1VSTE9lWitkbkZ5amtVN3ZJVkYyNkZyUzRITkxJcjVFYlBJUjhlWDFMdXhZUzdEaDFUM3FUclc= )

摘要：Android(安卓)是最流行的智能终端操作系统。当前越来越多攻击者把Android平台作为攻击对象,导致Android平台恶意代码泛滥,给用户造成经济损失、隐私泄露等诸多安全风险。因此Android平台的恶意代码检测已是智能终端安全领域一个亟待解决的问题。当前Android平台的恶意代码检测技术以静态检测和动态检测为主,大都只关注的应用的某一方面的特征,对恶意代码的特征考察不够全面,对不同类型恶意应用的检测能力有较大差别。为了解决这一问题,本文提出了基于多类特征的Android恶意代码检测技术。设计和实现了恶意代码检测系统,从应用的代码特征和行为特征两方面进行检测。研究内容可以归纳为以下几个方面:1.研究了通过应用代码相似度检测恶意代码的方法。为了检测相似的恶意应用,本文使用应用的代码相似度来检测诸如重打包、同族的恶意应用。通过提取应用的代码结构信息,以此为特征,提出了一种应用相似度计算方法,能够较好地反映应用之间的相似度,从而识别出相似的恶意应用。2.设计了基于操作路径的Android应用动态测试方法。为了解决传统工具测试的随机性导致的代码覆盖率低的问题,本文设计了基于操作路径的Android应用自动化测试方法。对应用的界面进行遍历,生成测试脚本,模拟用户操作,从而降低搜索空间,提高了动态测试效率。3.通过提取Android应用程序应用层行为特征,使用敏感API调用序列来检测恶意代码。结合API调用之间的依赖关系,提出基于API短序列的检测方法以及马尔可夫链的检测方法。实验验证了本文的方法能提高检测的准确率。4.建立Android恶意应用多类特征联合检测模型,能够对多种类型的恶意应用进行检测。首先用基于代码相似度的检测方法对相同或相似的恶意应用进行快速检测,然后使用基于应用行为的检测方法对其它恶意应用进行深度检测,提高了检测的覆盖面,保证了检测的准确性和快速性。
